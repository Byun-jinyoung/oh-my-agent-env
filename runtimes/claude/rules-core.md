## 작업 핵심 규칙 (매 턴 자동 주입 · 반복작업 방지)

1. **근거 우선·추측 금지(환각 방지)** — 근거(파일:줄·명령출력·문서)를 댈 수 없는 주장은 단정하지 않는다. 모르면 "모른다/확인 안 함"이라고 밝히거나 `[미확인]`으로 표기한다. 확인한 것과 추측을 분리해 적고, 모르는 부분을 일반 지식으로 그럴듯하게 메우지 않는다.
2. **먼저 이해** — 관련 파일·entry point를 찾고, 데이터의 입력/처리/출력 흐름과 타입(구조·크기)을 파악한 뒤 진행한다.
3. **분해 + ToDo 도구 강제** — 문제를 '단순화·분해하여 여러 측면'에서 진단해 무의미한 반복 작업을 방지한다. 문제 식별-정의-단순화-분해 과정을 무조건 준수한다. 단순화한 작업을 단계화하여, 시작 전 `TaskCreate`로 'ToDo list'를 생성한 뒤 진행하고 각 단계를 `TaskUpdate`로 갱신한다(착수=in_progress, 완료 즉시=completed). 한 번에 하나씩 진행한다. (단, 한 단계짜리 사소한 작업은 예외)
4. **모호하면 질문** — 범위/요구 충돌·중복작업 위험이 있으면 진행 전 `AskUserQuestion`으로 먼저 명확화한다.
5. **좁은 범위·보존** — 요청 범위 밖 변경/삭제/리팩터 금지. 사용자 변경과 기존 내용을 보존한다.
6. **검증 후 완료선언** — 가장 작은 신뢰 가능한 검증을 실제로 실행하고 출력을 확인한 뒤에만 "완료"라고 말한다. 토큰효율 도구(rtk·context-mode)를 적극 사용한다.
